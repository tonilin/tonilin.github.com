打工趣被 518 致(ㄔㄠ)敬(ㄒ一ˊ)了

May 29th, 2015 | Comments

打工趣是我們團隊 oSolve 花了很大的心血，研究出來適合打工族使用的 APP，並花了一個月開發和設計出來的。

在營運四個月後，也得到不錯的成績，各個平台 DAU 加起來約 8000 人，[DAU/MAU] 約為 0.25。

可是最近卻發現剛上架的 518 找打工 跟我們的 打工趣 相似度非常高，我想應該是被致(ㄔㄠ)敬(ㄒ一ˊ)了，下面來比較一下 打工趣 和 518 找工作 的雷同之處：

主要顏色

打工趣的顏色是 #35bbcf，518 找打工的顏色是 #14CEC9， 用 Delta-E Calculator 來計算兩個顏色的差異度，算出來的值是 17.9105，不管是眼睛看還是算出來的， 都可以看得出來兩個顏色非常相近。

工作列表頁面

  • 切換地區位置

為了讓使用者清楚得知道目前在看的是哪個地區，所以我們把地區切換按鈕放在左上角。同時找打工的使用者是不在乎你到底總共有多少筆資訊的， 重點是”新”資訊有多少，因此我們決定不顯示總共多少筆資料。518 致敬了好的部份，也把我們覺得不好該捨棄的部份加上去了。

  • 時薪、地點位置

除了工作內容以外，時薪和地點應該是使用者最在意的東西，所以標題下面馬上就接著時薪和地點。

  • 張貼時間

使用者會在意的其中一個因素，就是這個工作是不是最近張貼的，很重要，但是使用者只會看一眼，確定這一頁是最近一兩個禮拜的職缺， 所以放在右下角，才不會一直干擾使用者。

  • 收藏按鈕

當初本來只用星星，但發現星星放在這裡很醜，後來決定使用書籤 + 星號，讓這裡不會太單調。

  • 顯示已讀未讀

顯示已讀未讀是很重要的功能，可以節省找工作的時間，讓使用者知道上次看到哪裡，要從哪裡繼續看。 當初有兩個選擇，一個是目前的設計，一個是標題變成灰色。標題變成灰色不是一個好設計，這會讓使用者曾經在意過的工作變得很不明顯， “已額滿” 的工作才應該是用灰色字，讓使用者自動略過。這部份 518 可真的是無下限的致敬。

  • 「建議與問題回報」和「問題與回報」
  • 「評價打工趣」和「評價518找工作」
  • 「分享打工趣給朋友」和「分享518找工作給朋友」
  • 「有新工作通知我」和「新工作通知我」
  • 「關於打工趣」和「關於518找工作」
  • 「帳號」和「帳號」

工作內容

看到這個幾乎一模一樣的工作內容，真的心理五味雜陳啊～

履歷預覽功能

在送出前的履歷預覽功能，同樣有這個功能就算了，按鈕還一樣，只稍修改文案而已

新工作提示，形狀文案一模一樣

結論

打工市場和好的 UI 在台灣被漸漸重視是非常好的事情，但是這樣的致敬行為讓人覺得設計非常不被尊重。這麼大的公司致敬 一個不到五人的團隊，真的是見識到了 518 的下限。不過我們相信，好的東西才有(ㄔㄠ)敬(ㄒ一ˊ)的價值，你可以偷走我們的外殼， 可是沒辦法複製我們的靈魂，一個沒有靈魂的產品就只是個空殼而已。

開始創業，名字叫 oSolve

Apr 7th, 2014 | Comments

從出社會以後的第一間公司離職以後，就一直跟朋友說著要創業，我們有很多想法和想做的點子，卻一直都沒有行動，沒有行動的原因是覺得自己學的東西還不夠，因此決定繼續工作，學習經驗。但是有人跟我說，創業家永遠不會滿足於自己所學，永遠都有學不完的東西，於是我離開了公司，開始跟朋友一起創業。

台灣有非常多的接案公司，使用 Ruby on Rails 開發的卻很少，更可惜的是使用 Ruby 開發 Server 配合 Mobile APP 的公司卻連一家都找不到。就我所知，Ruby 有很多方便的 Gem，可以使開發 API Server 變得非常迅速而且有系統，而大部分的 APP 都是需要跟 Server 做溝通的，如果在 Server 端的開發有 Ruby 的加持，可以節省非常多時間，讓客戶的產品能在最快的時間上線。

於是我們公司提供了 Rails + Mobile APP 開發的解決方案，主要的服務項目是：

  1. Rails 網站開發
  2. iOS/Android APP 開發
  3. Web or APP 顧問
